Bestdori 文本转换器 - 功能使用文档

将纯文本剧本快速转换为 Bestdori 剧情编辑器可识别的 JSON 格式，简化创建对话卡片、配置说话角色、Live2D 动作及表情的流程

核心功能

  • 支持直接上传 .txt、.docx、.md 格式的剧本文件
  • 提供对话编辑器、Live2D 布局编辑器和动作/表情编辑器，通过拖拽方式配置剧情
  • 角色、服装、默认位置、引号都可以根据自己的习惯自定义
  • 所有自定义配置会保存在浏览器本地，也支持导出和导入配置文件

核心工作流

  1. 输入文本，把纯文本底稿整理成基础对话卡片
  2. 进入对话编辑器，把每张对话卡片和具体说话角色绑定起来
  3. 进入 Live2D 布局编辑器，控制角色的站位、登场、移动和退场
  4. 进入动作/表情编辑器，配置角色说话时的动作和表情
  5. 开始转换导出，复制或下载 JSON，再导入 Bestdori

其他注意事项

  • 右上角的问号按钮可以随时打开这份使用文档
  • 在页面右上角可以切换浅色和深色主题
  • 如果页面太挤，可以用 Ctrl + 滚轮 缩放页面
  • 如果你使用的是 Linux 系统的本地版本，退出程序时请点击右上角的电源按钮关闭服务

输入文本

这是项目的起点，用于将纯文本底稿转换为基础的对话卡片

文本要求

  • 将剧本粘贴到输入框时，每段对话之间必须至少保留一个空行
  • 如果没有空行，程序会把所有文本挤到同一个对话卡片里
  • 你可以直接粘贴文本，也可以上传 .txt、.docx、.md 文件

格式化文本

  • 点击【格式化文本】后，程序会自动清理多余空白符
  • 程序也会自动调整对话之间的空行间距
  • 建议每次编辑之前先点一次格式化文本

编辑器概述

这里整理三个编辑器共用的操作入口和通用说明

通用操作

  • 三个编辑器里都可以直接拖拽卡片调整顺序
  • 在编辑器中都可以使用 Ctrl+Z 撤销和 Ctrl+Y 恢复
  • 修改完成后记得点击底部的【保存】，保存当前编辑内容

编辑器卡片分组

  • 当对话卡片、布局卡片或动作表情卡片太多，导致编辑器浏览卡顿时，可以打开【启用编辑器卡片分组】
  • 开启后，编辑器会按分组显示卡片，浏览大量内容时会更顺畅一些
  • 如果你想直接查看完整的连续卡片列表，也可以再把这个选项关掉

项目进度文件导入导出

  • 三个编辑器底部都有【导入进度】和【导出进度】按钮，用来保存和继续编辑当前项目进度
  • 导出的项目进度文件会保存当前的对话、布局、动作和表情配置，方便分几次继续做
  • 从对话编辑器、Live2D 布局编辑器或动作表情编辑器导入都一样，读取的都是同一种项目进度文件
  • 你可以在哪个编辑器里顺手导入都可以，导入后再继续修改当前工程就行

对话编辑器

在这里将文本卡片和具体的说话角色进行绑定

界面和分配方式

  • 左侧是对话卡片列表，右侧是角色列表
  • 同一张对话卡片可以添加多个说话角色
  • 直接把右侧角色头像拖到左侧卡片上，就能完成说话人分配
  • 从卡片上反向拖出角色，可以移除这个角色
  • 也可以先点击选中对话卡片，再拖拽右侧角色，实现固定卡片的角色分配

常用功能

  • 点击角色头像右侧的图钉，可以把常用角色固定在列表顶部
  • 打开多选模式后，可以一次性勾选多张卡片，再统一点击角色批量分配
  • 如果需要修改台词，可以打开编辑模式，编辑目标卡片上的文本
  • 点击底部的【恢复默认】会清空当前对话卡片上的说话角色分配

自动识别对话卡片的说话角色

  • 程序会识别“角色名 + 冒号 + 对话内容”的格式，对话内容有没有引号都可以
  • 如果配置管理里已经有这个角色的名称和 ID 映射，程序会自动为这个对话卡片添加对应的说话角色
  • 所以你也可以先在自己的文本里，给需要添加说话角色的内容开头加上角色名字和冒号，这样就能省去后面手动添加说话人的操作
  • 例如：户山香澄：kirakira-dokidoki，这样 kirakira-dokidoki 这个对话卡片会自动以户山香澄作为说话角色

Live2D 布局编辑器

用于控制角色在屏幕上的站位、登场与退场

页面布局

  • 左侧是对话卡片和布局卡片列表
  • 右侧是角色列表，可以从这里把角色拖进列表

布局卡片

  • 你可以在对话之间插入布局卡片，控制角色在屏幕上的站位、登场与退场
  • 布局卡片会插在对话之间，用来控制角色什么时候登场、移动和退场
  • 布局卡片可以控制模型的位置、服装和出场类型

移动布局和自定义位置

  • 点击底部的【快速布局】后，程序会自动在角色首次出现的对话卡片前面添加相应的布局卡片
  • 当同一个模型在登场后到退场前之间再次添加相同Live2D模型，程序会自动识别成移动布局
  • 点击连线可以切换退场布局或移动布局
  • 点击自定义位置配置，可以单独修改 Live2D 移动时的起点和终点坐标
  • 点击底部的【清空布局】会移除当前卡片列表里的所有布局卡片

动作/表情编辑器

为角色的 Live2D 模型配置说话时的神态和动作

页面布局

  • 左侧是卡片列表，点击设置动作表情可以展开角色动作表情分配区
  • 右侧是动作和表情资源栏，可以从这里拖拽到左侧角色状态栏
  • 资源栏里可选的动作和表情配置选项是根据当前在场角色来的
  • 比如当前在场的是户山香澄和花园多惠，那么资源栏里就会显示与她们相关的动作和表情选项

动作表情预览

  • 点击【跳转 Live2D 浏览器】后，程序会根据你已登场的Live2D服装，在 Bestdori 中打开对应的 Live2D 动作表情预览页
  • 推荐在电脑上分屏操作，一边看预览，一边填动作和表情参数

快速填写

  • 在动作或表情输入框里输入前缀，可以快速筛选动作/表情选项
  • 可以使用下拉菜单直接选择表情，减少键盘输入
  • 快速填充选项也支持自定义添加，你可以按自己的习惯补充常用项

临时添加和恢复默认

  • 如果当前资源栏里没有你要的动作或表情，也可以先在下方手动添加临时动作 ID 或临时表情 ID
  • 临时添加的项目会立刻出现在当前资源栏里，方便直接拖拽到左侧角色状态栏
  • 这些临时项目只在当前编辑器窗口里生效，关闭模态框后就会消失
  • 如果你想长期保存这些动作或表情配置，需要到【动作/表情设置】页面里添加
  • 点击底部的【恢复默认】会清空当前时间线里的动作和表情配置

处理选项

管理旁白名称和文本首尾引号的处理方式

基础设置

  • 如果一个对话卡片没有分配任何角色，它会被视为旁白
  • 默认旁白名称是一个空格，你可以在基础设置里修改
  • 自动添加结尾空格和自动添加换行前空格都在这里设置
  • 这两项用来控制对话停留时间和多行文本的分段延时

引号处理

  • 可以自动去除对话卡片首尾匹配的引号
  • 默认自动勾选了多种双引号，你可以自行添加自定义引号
  • 如果你希望生成后的剧情保留文本首尾引号，可以取消双引号处理选项

配置管理概述

这里整理配置相关页面共用的保存方式和默认配置说明

保存和恢复默认

  • 角色 ID、服装、位置、动作和表情这些配置页修改后，都要记得点击对应页面底部的【保存】
  • 只有点过保存，这些修改才会真正写进本地配置里
  • 如果你先点了【恢复默认】，后面也要再点一次【保存】，恢复结果才会真正生效

修改 config.yaml 默认配置

  • 如果想要更改默认映射的角色 ID，可以打开 config.yaml 文件，把角色名后面对应的 ID 数字改成你想要的就可以了
  • 在 config.yaml 里你也可以修改其它默认配置选项
  • 修改并保存后，需要重新启动程序才会生效

导入导出和清除缓存

  • 所有自定义配置会自动保存在浏览器本地
  • 你可以把当前配置导出成一个 JSON 文件进行备份
  • 导出时会包含角色名称和 ID 映射、自定义引号、服装配置、位置配置、动作和表情设置、快速填充选项
  • 基础设置里的自动添加结尾空格和自动添加换行前空格也会一起导出
  • 也可以导入之前导出的配置文件继续使用
  • 如果遇到无法解释的奇怪问题，可以使用清除缓存把应用恢复到初始状态

角色ID映射配置

通过角色 ID 管理角色名称和头像显示

角色 ID 机制

  • 这里用来管理角色名称、角色 ID 和头像映射
  • 程序通过角色 ID 控制显示的头像和名称
  • 你可以新增一个自定义角色名称，再填入一个现有角色的 ID
  • 这样即使是 Bestdori 还没有官方头像的新角色，也能先借用别人的头像来辨认

常见用法

  • 适合用来补充别名、修正角色名识别和添加自定义角色
  • 可以导出配置做备份

服装配置

给角色补充内置之外的服装并设置默认服装

服装来源

  • 这里可以给角色补充可用服装和默认服装
  • 程序内置了最常用的几套服装
  • 如果需要其它服装，可以点击【浏览数据库】跳转到 Bestdori 服装库
  • 在数据库里找到想要的服装后，复制它的名称回到这里添加即可

使用建议

  • 服装名称示例：014_casual_2023
  • 在 Bestdori 数据库里可以用 Ctrl + F 快速搜索角色 ID 来找想要的角色的Live2D服装
  • 复制后回到服装配置管理页面添加即可

位置配置

修改角色出现在左侧、居中、右侧时的默认屏幕坐标

默认位置

  • 这里用来修改角色在左侧、居中、右侧时的默认坐标
  • 开启自动分配位置模式后，添加布局卡片时会自动按左侧、居中、右侧的顺序循环给角色分配默认站位
  • 这里可以直接修改左侧、居中、右侧三种默认偏移坐标
  • 保存之后，新添加的布局卡片会优先使用这里的新坐标

动作/表情设置

补录特殊服装专属的动作和表情代码

适合什么情况

  • 这里可以自行添加缺少的动作和表情配置选项
  • 内置的动作表情配置选项不一定完整，你可以根据需要自行添加
  • 自行添加的动作表情配置，它们就会固定出现在动作/表情编辑器里，不受登场角色限制

转换导出

完成所有编排后，将工程输出为 Bestdori 格式

开始转换

  • 点击【开始转换】按钮后，下方会显示生成好的 JSON 代码
  • 建议在转换前先确认处理选项里面的设置，引号处理是否正确勾选

跳转

  • 点击【跳转到 Bestdori】后，程序会自动复制生成的 JSON
  • 随后会直接打开 Bestdori 的剧情编辑器页面

导入 Bestdori

  • 在 Bestdori 剧情编辑器页面点击导入
  • 把刚才复制的 JSON 粘贴进去，就能完成整段剧情的构建

结果合并

适合把分开做的章节结果重新拼回一份完整文件

适合什么场景

  • 可以合并两种文件：Bestdori 结果文件，或者项目进度文件
  • 适合把长文本剧情拆成几段分别配置，再在这里重新拼成一份完整文件
  • 这样可以先一段一段导入 Bestdori 调试，最后再统一合并完整结果
  • 把分章节做好的多个结果文件合成一个完整文件
  • 把多个项目进度文件合并成一条连续时间线
  • 调整文件顺序后再统一导出
  • 适合把分开制作的章节、片段或版本重新拼成一份完整结果

使用建议

  • 同一次合并里必须使用同一种文件，不能把 Bestdori 结果文件和项目进度文件混在一起
  • 合并前先按章节顺序排好列表
